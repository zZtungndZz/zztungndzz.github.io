Kế hoạch triển khai

Kế hoạch này sử dụng phương pháp phát triển linh hoạt (Agile), chia dự án thành các giai đoạn (Phase) lớn, và trong mỗi giai đoạn sẽ có các Sprint nhỏ (thường là 2 tuần) để quản lý công việc cụ thể.

Thành phần Đội ngũ & Vai trò

  • Quản lý Dự án (1 Người) Chịu trách nhiệm về tầm nhìn sản phẩm. Xác định và ưu tiên hóa các tính năng trong backlog. Làm cầu nối giữa đội ngũ phát triển và các bên liên quan. Quản lý tiến độ, rủi ro và ngân sách dự án.
  • Designer (1 người) Chịu trách nhiệm về toàn bộ trải nghiệm người dùng (UX) và giao diện người dùng (UI).
    Công cụ: Figma, Adobe XD.
    Nhiệm vụ: Wireframe, Mockup, Design System, Prototype.
  • Developer 1 (Backend & Infrastructure Focus) Chịu trách nhiệm chính về Server, CSDL, API, và hạ tầng.
    Công nghệ: .NET Core, MongoDB, Redis, DevOps.
  • Developer 2 (Frontend & Agent Focus) Chịu trách nhiệm chính về Dashboard Frontend và Agent trên client.
    Công nghệ: Next.js, Windows Service (C#), SQLite.
  • Tester/QA (1 người) Chịu trách nhiệm đảm bảo chất lượng cho toàn bộ sản phẩm.
    Công cụ: Jira/Trello (quản lý bug), Postman (test API), Test Case Management tools.
    Nhiệm vụ: Lập kế hoạch kiểm thử, viết test case, kiểm thử thủ công, kiểm thử API, báo cáo lỗi.

Tổng Thời gian Dự kiến: ~12 Tuần (~3 tháng) cho phiên bản 1.0

Giai đoạn 0: Nền tảng & Thiết kế (Sprint 0) - (2 Tuần)

Mục tiêu: Chuẩn bị mọi thứ cần thiết trước khi viết code tính năng. Đảm bảo toàn đội có chung một tầm nhìn.

Chi tiết công việc
Hoạt động Người thực hiện Kết quả đầu ra (Deliverables)
1. Thiết kế UX/UI Designer - Wireframe cho các màn hình chính của MVP.
- Mockup (thiết kế chi tiết) cho màn hình Login, Dashboard tổng quan, Danh sách người dùng.
- Xây dựng bộ Design System cơ bản trong Figma (màu sắc, font, button...).
2. Thiết lập Môi trường & Hạ tầng Dev 1 (Backend) - Tạo repository trên Git (Server, Frontend, Agent).
- Thiết lập CI/CD cơ bản (ví dụ: GitHub Actions).
- Cài đặt và cấu hình MongoDB, Redis trên môi trường dev.
- Dựng project ASP.NET Core, cấu trúc folder, thiết lập logging.
3. Thiết lập Môi trường Frontend/Agent Dev 2 (Frontend/Agent) - Dựng project Next.js, tích hợp thư viện UI (MUI/Antd).
- Dựng project Windows Service C# cơ bản, thử nghiệm chạy dưới dạng dịch vụ.
4. Lập kế hoạch Kiểm thử Tester - Tạo project trên Jira/Trello.
- Viết Test Plan tổng thể.
- Chuẩn bị các Test Case ban đầu cho MVP.
5. Họp Toàn đội Tất cả - Thống nhất về thiết kế, kiến trúc, và quy trình làm việc (cách đặt tên nhánh git, quy trình review code, báo cáo bug...).

Giai đoạn 1: Xây dựng MVP (Sản phẩm Khả dụng Tối thiểu) - (3 Tuần)

Mục tiêu: Hoàn thành một luồng end-to-end: Cài Agent -> Agent gửi dữ liệu cơ bản -> Admin xem được dữ liệu trên Dashboard.

Chi tiết công việc
Hạng mục Người thực hiện Chi tiết công việc
Agent (MVP) Dev 2 - Hoàn thiện bộ cài đặt đơn giản, chạy dưới dạng Windows Service.
- Xây dựng tính năng giám sát ứng dụng/web và theo dõi thời gian hoạt động.
- Tích hợp cơ chế cache (SQLite) và gửi dữ liệu an toàn (HTTPS) lên Server.
Server (MVP) Dev 1 - Xây dựng API xác thực cho Agent và Admin (Login).
- Xây dựng API tiếp nhận dữ liệu từ Agent, đẩy vào Redis và worker xử lý lưu vào MongoDB.
- Xây dựng API cung cấp dữ liệu cơ bản cho Dashboard (danh sách người dùng, log hoạt động...).
Dashboard (MVP) Dev 2 - Xây dựng trang Login, layout chính.
- Xây dựng trang Dashboard tổng quan, danh sách người dùng.
- Xây dựng trang chi tiết người dùng, hiển thị log hoạt động đã thu thập.
Kiểm thử Tester - Liên tục: Test API bằng Postman ngay khi Dev 1 hoàn thành. Test từng tính năng của Agent và Dashboard.
- Cuối giai đoạn: Kiểm thử tích hợp toàn bộ luồng MVP.
Thiết kế Designer - Hỗ trợ các Dev làm đúng thiết kế.
- Bắt đầu thiết kế cho các tính năng của Giai đoạn 2.

Kết quả Giai đoạn 1: Một phiên bản hoạt động được, chứng minh kiến trúc là khả thi.

Giai đoạn 2: Mở rộng Tính năng Cốt lõi - (2 Tuần)

Mục tiêu: Thêm các tính năng giám sát quan trọng và hệ thống cảnh báo, quản lý.

Chi tiết công việc
Hạng mục Người thực hiện Chi tiết công việc
Agent Dev 2 - Thêm tính năng Chụp ảnh màn hình (theo cấu hình).
- Thêm tính năng Keylogging và Giám sát hoạt động file.
- Tích hợp cơ chế nhận chính sách từ Server (Policy).
Server Dev 1 - Xây dựng API quản lý chính sách (Policy Engine).
- Xây dựng hệ thống cảnh báo (Alerting Engine) cơ bản (quét CSDL, gửi email).
- Tối ưu hóa việc lưu trữ file ảnh (có thể dùng S3 hoặc lưu trong GridFS của Mongo).
Dashboard Dev 2 - Xây dựng giao diện cho Policy Editor (để tạo/sửa chính sách).
- Xây dựng giao diện xem lại ảnh chụp màn hình, xem log gõ phím.
- Xây dựng giao diện quản lý và hiển thị cảnh báo.
Kiểm thử Tester - Test chuyên sâu từng tính năng giám sát mới.
- Test luồng tạo Policy và kiểm tra xem Agent có áp dụng đúng không.
- Test tính đúng đắn của hệ thống cảnh báo.
Thiết kế Designer - Thiết kế giao diện cho Policy Editor, trang xem ảnh, trang quản lý cảnh báo.
- Đảm bảo UX/UI nhất quán.

Giai đoạn 3: Tính năng Nâng cao & Hoàn thiện - (3 Tuần)

Mục tiêu: Bổ sung các tính năng "pro" và cải thiện trải nghiệm người dùng.

Chi tiết công việc
Hạng mục Người thực hiện Chi tiết công việc
Agent Dev 2 - Hoàn thiện cơ chế tự động cập nhật trong im lặng.
- Thêm tính năng giám sát USB.
Server Dev 1 - Xây dựng hệ thống báo cáo (Reporting Engine) - xuất CSV/PDF.
- Xây dựng tính năng Audit Log cho quản trị viên.
Dashboard Dev 2 - Xây dựng giao diện cho tính năng Báo cáo (chọn tiêu chí, xuất file).
- Xây dựng trang xem Audit Log. Cải thiện, tối ưu các biểu đồ.
Kiểm thử Tester - Test kỹ lưỡng tính năng tự động cập nhật.
- Test tính chính xác của dữ liệu trong các báo cáo được xuất ra.
- Bắt đầu thực hiện Performance Testing (kiểm thử hiệu năng) cơ bản.
Thiết kế Designer - Thiết kế giao diện cho trang báo cáo, audit log.
- Rà soát lại toàn bộ sản phẩm để cải thiện UX.

Giai đoạn 4: Ổn định & Phát hành (Stabilization) - (2 Tuần)

Mục tiêu: Không thêm tính năng mới. Tập trung 100% vào sửa lỗi, tối ưu hiệu năng và chuẩn bị cho việc triển khai thực tế.

Chi tiết công việc
Hoạt động Người thực hiện Chi tiết công việc
Sửa lỗi (Bug Fixing) Dev 1, Dev 2 - Xử lý toàn bộ các bug còn tồn đọng trong Jira/Trello theo thứ tự ưu tiên.
Tối ưu hóa Dev 1, Dev 2 - Server: Tối ưu các câu lệnh truy vấn CSDL chậm, giảm bộ nhớ sử dụng.
- Agent: Giảm mức tiêu thụ CPU/RAM.
- Dashboard: Tăng tốc độ tải trang, tối ưu rendering.
Kiểm thử Toàn diện Tester - Thực hiện kiểm thử hồi quy (Regression Testing) toàn bộ hệ thống.
- Kiểm thử bảo mật (Security Testing) ở mức cơ bản (ví dụ: kiểm tra lỗ hổng XSS, SQL Injection...).
- Phối hợp thực hiện UAT (User Acceptance Testing) với một nhóm người dùng thử.
Hoàn thiện Tài liệu Tất cả - Viết tài liệu hướng dẫn cài đặt Agent.
- Viết tài liệu hướng dẫn sử dụng Dashboard cho Admin.

Rủi ro và Giải pháp

  • Rủi ro: Agent bị các phần mềm Anti-virus nhận diện là mã độc. Giải pháp: Nghiên cứu các kỹ thuật để Agent hoạt động "thân thiện" hơn, mua chứng chỉ số (Code Signing Certificate) để ký vào file cài đặt, chuẩn bị tài liệu hướng dẫn "whitelist" Agent.
  • Rủi ro: Hiệu năng hệ thống không đáp ứng khi có nhiều Agent cùng hoạt động. Giải pháp: Thiết kế kiến trúc Server có khả năng mở rộng ngay từ đầu (sử dụng queue, stateless services). Dành thời gian ở Giai đoạn 4 để đo lường và tối ưu.
  • Rủi ro: Đội ngũ Frontend/Agent (Dev 2) bị quá tải vì phải làm cả 2 mảng. Giải pháp: Dev 1 cần linh hoạt hỗ trợ Dev 2, ví dụ như giúp tích hợp API cho Dashboard. Ưu tiên các tính năng, có thể dời bớt tính năng phức tạp sang phiên bản sau.
  • Rủi ro: Yêu cầu về tính năng thay đổi liên tục. Giải pháp: Tuân thủ quy trình Agile. Các thay đổi lớn chỉ được xem xét vào đầu mỗi Giai đoạn. Các thay đổi nhỏ có thể được điều chỉnh trong các Sprint.
